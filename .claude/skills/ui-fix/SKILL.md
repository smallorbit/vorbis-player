# UI Fix Skill
When making UI/CSS changes:
1. Read the relevant component files first to understand current layout structure
2. Describe 2-3 possible approaches BEFORE implementing anything
3. Implement the MINIMAL change that addresses the request
4. Do NOT add extra spacers, wrappers, or layout restructuring unless explicitly requested
5. After the edit, run `npm run build` to verify
6. If the approach doesn't work, REVERT completely before trying a different approach — don't layer fixes on top of broken changes
